# 0.0.2 Documentation: Layout Spec

## Overview

The documentation hub inside `/styleguide/docs` uses a dedicated layout and typography system. This spec defines the CSS utilities (`@kol/ui/css/docs.css`) and React wrappers (`DocsHeader`, `DocsLayout`, `DocsArticle`, etc.) that replace ad-hoc overrides while keeping the layout aligned to the 304 / 680 / 192 grid.

## Tokens

### Typography

| Token/Class     | Font                                       | Size / Leading | Color token                  | Usage                                  |
|-----------------|--------------------------------------------|----------------|------------------------------|----------------------------------------|
| `.docs-title`   | Right Grotesk Narrow                       | 48px / 1.05    | `color-mix(... 88%)`         | Page title & `# H1` headings           |
| `.docs-subtitle`| Mono                                       | 14px / 1.6     | `color-mix(... 48%)`         | Single-line descriptions               |
| `.docs-meta`    | Mono uppercase                             | 11px / 1.6     | `color-mix(... 64%)`         | Metadata pairs (Version, Status, etc.) |
| `.docs-heading` | Right Grotesk Narrow                       | 24px / 1.35    | `color-mix(... 88%)`         | H2 within docs                         |
| `.docs-heading-sm` | Right Grotesk Narrow                   | 18px / 1.45    | `color-mix(... 80%)`         | H3 within docs                         |
| `.docs-body` *(inherited via `.docs-article`)* | Sans-serif  | 14px / 1.7     | `color-mix(... 48%)`         | General body copy                      |
| `.docs-list`    | Sans-serif                                 | inherit / 1.38 | `color-mix(... 48%)`         | Dense lists; add `.docs-list--ordered` for numerals |
| `.docs-codeblock` | Mono                                     | 13px / 1.4     | `color-mix(... 64%)` on `bg-fg-02/40` | Code samples & directory trees |
| `.docs-callout` | Sans-serif                                 | inherit / 1.6  | `color-mix(... 64%)`         | Blockquotes, callouts                  |

### Layout

- `.docs-layout` – 1320px max-width grid, 40px outer gutter, collapses to a column stack below 1024px.
- `.docs-column-nav` & `.docs-column-navSticky` – 304px column for sticky summary/navigation.
- `.docs-column-main` – 680px content column; automatically applies the typographic stack above.
- `.docs-column-toc` – 192px sticky table-of-contents column.
- `.docs-folderbar` – Horizontal pill row for folder navigation.
- `.docs-breadcrumb` – Uppercase mono breadcrumb trail aligned with the main grid.

## React Wrappers

```jsx
import {
  DocsHeader,
  DocsLayout,
  DocsNavColumn,
  DocsMainColumn,
  DocsTocColumn,
  DocsArticle
} from '@kol/ui/docs'
```

- **`DocsHeader`** – Renders `.docs-title`, `.docs-subtitle`, and optional meta pairs.
- **`DocsLayout`** – Applies the 3-column grid. Use composition for the nav/main/toc columns.
- **`DocsNavColumn`** – Handles sticky summary cards, filters, and collection groups.
- **`DocsArticle`** – Wraps markdown sections with the docs type ramp.
- **`DocsTocColumn`** – Sticky right column listing headings / quick actions.

## Parser Requirements

To match the spec, markdown renderers should map tokens as follows:

| Markdown         | Output Element / Class                    |
|------------------|-------------------------------------------|
| `# Title`        | `.docs-title` (handled in `DocsHeader` or markdown H1) |
| `## Heading`     | `<h2>` inside `.docs-article`             |
| `> Callout`      | `<blockquote class="docs-callout">`       |
| Inline metadata (`**Label:** Value`) | `<div class="docs-meta"><span><strong>Label</strong> Value</span></div>` |
| Fenced code      | `<pre class="docs-codeblock"><code>...</code></pre>` |
| Lists            | `<ul class="docs-list tight">` or `<ol class="docs-list docs-list--ordered tight">` |
| Horizontal rule (`---`) | `<Divider className="w-full opacity-60">`  |

## Usage Guidelines

1. **Import styles** – ensure `@kol/ui/css/docs.css` is loaded (already imported by `apps/web/src/index.css`).
2. **Wrap content** – use `DocsLayout` + column components for every docs page.
3. **Source data** – populate folder tags, summary cards, and collection lists from `system-metrics.json`.
4. **Prefer tokens** – do not set inline colors or fonts; rely on the utilities above.
5. **Responsive behavior** – ensure mobile stacks maintain 24px gutters and collapse sticky columns gracefully.
