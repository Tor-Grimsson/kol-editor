## Spacing System Reference

### Design Tokens (from Tailwind config)
```
4px   → 1 unit   (p-1, m-1, gap-1)
8px   → 2 units  (p-2, m-2, gap-2)
12px  → 3 units
16px  → 4 units
20px  → 5 units
24px  → 6 units
32px  → 8 units
40px  → 10 units
48px  → 12 units
64px  → 16 units
80px  → 20 units
96px  → 24 units
```

### Standard Spacing Patterns
- **Component internal spacing**: 4-8px (p-1 to p-2)
- **Between elements**: 8-16px (gap-2 to gap-4)
- **Between sections**: 24-48px (py-6 to py-12)
- **Page margins**: 32px (px-8)
- **Max content width**: 1400px

## Findings

### Critical Issues

#### 1. Inconsistent Hero Section Spacing
**Location**: `/Users/biskup/git/kolkrabbi/kolkrabbi-monorepo/apps/web/src/components/sections/collections/CollectionHero.jsx:23`
**Current**: `pt-24 pb-16 mt-24`
**Expected**: `pt-24 pb-24 mt-24` or standardize hero bottom padding
**Why it matters**: Hero sections should have balanced top/bottom padding. This uses pb-16 (64px) while most hero sections use pb-24 (96px) or pt-24 pb-24 for symmetry.
**Pattern Found**:
- FoundryOverview hero: `pt-24 pb-24 lg:pt-36 lg:pb-36 mt-24` (line 55)
- FoundryLicensing hero: `pt-24 pb-24 lg:pt-36 lg:pb-36 mt-24` (line 71)
- CollectionHero: `pt-24 pb-16 mt-24` (inconsistent)

#### 2. Inconsistent Section Vertical Spacing
**Location**: Multiple pages - mixed use of py-12, py-16, and py-24
**Current**: No clear pattern
**Expected**: Consistent section spacing (py-16 for standard sections, py-24 for major sections)
**Why it matters**: Creates uneven visual rhythm across pages
**Examples**:
- CollectionsOverview sections use py-16 consistently (lines 147, 160) ✓
- FoundryOverview sections use py-16 consistently (lines 81, 126, 133, 158) ✓
- Illustrations/Logomarks pages use py-12 in main wrapper (line 70) - inconsistent with overview pages
- Motion Graphics uses pb-8 (line 213) and pb-24 (line 227) - very inconsistent

#### 3. Unbalanced Vertical Spacing in Motion Graphics Page

## Testing Method

1. **Visual Inspection**
   - Navigate to each page
   - Use browser DevTools to inspect spacing
   - Compare similar sections for consistency

2. **Measure Spacing**
   - Use DevTools computed styles
   - Note actual pixel values
   - Compare against design tokens

3. **Responsive Testing**
   - Test at breakpoints: 320px, 768px, 1024px, 1400px, 1920px
   - Verify spacing scales appropriately
   - Check for overflow or cramped layouts

4. **Document Findings**
   - Screenshot inconsistencies
   - Note file path and line number
   - Propose fix with specific values
