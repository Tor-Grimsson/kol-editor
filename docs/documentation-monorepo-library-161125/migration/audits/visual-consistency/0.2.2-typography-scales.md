## Common Issues to Check

### Font Family Issues
```jsx
// BAD - Using product font in UI
<h2 className="font-['TGMalromur']">Section Title</h2>

// GOOD - Using design system font
<h2 className="kol-heading-lg">Section Title</h2>

// OK - Product font in specimen context
<div className="specimen-display font-['TGMalromur']">
  Specimen Text
</div>
```

### Heading Hierarchy Issues
- [ ] Skipping heading levels (h1 → h3)
- [ ] Multiple h1 elements on one page
- [ ] Inconsistent heading sizes for same level
- [ ] Wrong font-stretch (tight vs narrow vs compact)

### Text Scale Issues
- [ ] Body text too small (<14px)
- [ ] Body text too large (>18px for body)
- [ ] Inconsistent paragraph spacing
- [ ] Poor line height (too tight or too loose)

### Class Naming Issues
- [ ] Using legacy classes (kol-body, kol-h1, etc.)
- [ ] Wrong class for context (display instead of heading)
- [ ] Missing responsive scaling (helpers are fixed)

### Responsive Issues
- [ ] Text too large on mobile
- [ ] Text too small on desktop
- [ ] Poor scaling between breakpoints
- [ ] Overflow on small screens

## Typography Hierarchy Best Practices

### Standard Page Structure
```jsx
<main>
  <section>
    {/* Display for hero/marquee */}
    <h1 className="kol-display-lg">Page Title</h1>
    <p className="kol-mono-text text-fg-64">Subtitle</p>
  </section>

  <section>
    {/* Headings for content */}
    <h2 className="kol-heading-lg">Section Title</h2>
    <p className="kol-text-md">Body paragraph...</p>

    <h3 className="kol-heading-md">Subsection</h3>
    <p className="kol-text-md">More content...</p>
  </section>
</main>
```

### Card Typography Pattern
```jsx
<div className="card">
  <span className="kol-label-mono-xs uppercase">Category</span>
  <h3 className="kol-heading-sm">Card Title</h3>
  <p className="kol-mono-sm text-fg-64">Description</p>
</div>
```

### Hero Pattern
```jsx
<section className="hero">
  <span className="kol-label-mono-xs uppercase text-fg-64">Label</span>
  <h1 className="kol-display-section">Large Hero Title</h1>
  <p className="kol-text-md text-fg-64">Hero description</p>
</section>
```

## Testing Method

1. **Visual Inspection**
   - Review each page for hierarchy clarity
   - Check font families are correct (design system vs product)
   - Verify sizes feel appropriate
   - Look for old class names

2. **DevTools Inspection**
   - Check computed font-size values
   - Verify font-family in use
   - Check line-height values
   - Inspect font-weight and font-stretch

3. **Hierarchy Validation**
   - Use browser outline view (h1, h2, h3...)
   - Ensure logical hierarchy
   - No skipped levels

4. **Readability Testing**
   - Text should be readable at normal viewing distance
   - Good contrast between text and background
   - Appropriate line length (<80 characters)
