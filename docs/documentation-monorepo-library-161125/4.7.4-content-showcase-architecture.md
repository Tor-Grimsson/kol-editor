# 4.7.4 Content Showcase Architecture

## Overview

Collections and Foundry share a common architectural pattern for showcasing and browsing content. Both sections use similar components, layouts, and navigation patterns to provide consistent user experiences for exploring different types of creative work.

## Shared Architecture

### Common Page Structure

Both Collections and Foundry follow this hierarchy:

```
Overview/Landing Page
  ↓
Featured Items Carousel
  ↓
Filtered Grid/Index
  ↓
Individual Item Pages
```

### Shared Components

#### OverviewHero
**Location**: `packages/ui/src/molecules/OverviewHero.jsx`

**Purpose**: Hero section for overview/landing pages

**Used In**:
- CollectionsOverview (`/collections`)
- FoundrySpecimens (`/foundry/specimens`)
- Future: All Typefaces, Licensing

**Props**:
- `badge` - Badge text (e.g., "Collections", "Specimens")
- `badgeVariant` - Pill variant (default: "inverse")
- `title` - Main heading
- `description` - Body text
- `categories` - Array of category pills

**Pattern**:
```jsx
<OverviewHero
  badge="Collections"
  title="Kolkrabbi Collections"
  description="A curated showcase..."
  categories={['Illustrations', 'Logo Marks', 'Visual Experiments']}
/>
```

#### FeaturedItemsCarousel
**Location**: `packages/ui/src/organisms/FeaturedItemsCarousel.jsx`

**Purpose**: Rotating carousel for featured items with auto-rotation

**Used In**:
- CollectionsOverview - showcases illustrations, logomarks, motion graphics
- FoundrySpecimens - showcases specimen previews with live card renders

**Features**:
- Auto-rotation (configurable interval)
- Manual navigation controls
- Counter display (X / Y items)
- Custom content rendering via `renderContent` prop
- Metadata display (title, subtitle, description)
- Call-to-action links

**Props**:
- `items` - Array of items to display
- `renderContent` - Function to render item preview: `(item) => ReactNode`
- `autoRotate` - Enable auto-rotation (default: true)
- `interval` - Rotation interval in ms (default: 5000)
- `counterLabel` - Counter section label (e.g., "Featured Work")

#### Filter Components

Both sections use similar filter patterns:

**CollectionFilters** (`apps/web/src/components/sections/collections/CollectionFilters.jsx`):
- Filter by category, type, year
- Expandable/collapsible interface
- Tag-based filter selection
- Active filter count

**SpecimenFilters** (`apps/web/src/components/sections/foundry/SpecimenFilters.jsx`):
- Filter by typeface, category
- View mode toggle (Hubs/All/By Typeface)
- Similar UI pattern to CollectionFilters
- Tag-based filter selection

**Shared Pattern**:
- Expandable with filter icon button
- Tag components for filter options
- Active filter highlighting
- "Clear all" functionality
- Real-time filtering with useMemo

#### QuickLinksGrid
**Location**: `packages/ui/src/organisms/QuickLinksGrid.jsx`

**Purpose**: Grid of link cards for navigation to subcategories

**Used In**:
- CollectionsOverview - links to Illustrations, Logomarks, Motion Graphics

**Potential Future Use**:
- Foundry overview page linking to Specimens, Typefaces, Licensing

#### FoundryCTA
**Location**: `packages/ui/src/organisms/FoundryCTA.jsx`

**Purpose**: Bottom call-to-action section with heading and link

**Used In**:
- CollectionsOverview
- FoundrySpecimens
- Individual collection pages

**Pattern**: Consistent CTA placement at bottom of overview pages

### TypefaceCard Component
**Location**: `packages/ui/src/atoms/foundry/TypefaceCard.jsx`

**Purpose**: Card component for displaying typeface specimens or collections

**Used In**:
- FoundrySpecimens grid
- Specimen hub grids
- Can be adapted for other card-based grids

**Features**:
- Hover states
- Font family preview
- Subtitle and description
- Consistent sizing in grids

## Architecture Comparison

### Collections
```
CollectionsOverview (/collections)
├── OverviewHero
├── FeaturedItemsCarousel (illustrations, logomarks, motion)
├── QuickLinksGrid (navigation cards)
└── FoundryCTA

Individual Collections (/collections/illustrations)
├── Hero section
├── CollectionFilters
├── Grid of items
└── FoundryCTA
```

### Foundry Specimens
```
FoundrySpecimens (/foundry/specimens)
├── OverviewHero
├── FeaturedItemsCarousel (specimen previews)
├── SpecimenFilters
├── Grid of TypefaceCards
└── FoundryCTA

Specimen Hub (/specimen/[typeface])
├── Back link
├── Typeface introduction
├── Collection cards
└── Links to specimens

Individual Specimen (/specimen/[typeface]/[pattern])
├── GridOverlay wrapper
├── Specimen cards
└── ViewToggle for grid
```

## Data Structure Patterns

### Collections Data
**Location**: `apps/web/src/data/`
- `illustrations.js`
- `logomarks.js`
- Inline motion graphics data

**Structure**:
```javascript
{
  id: number,
  name: string,
  type: string,
  description: string,
  category: string,
  featured: boolean,
  // Type-specific fields (illustrationName, logoName, videoUrl)
}
```

### Specimens Data
**Location**: `apps/web/src/data/foundry/specimens.js`

**Structure**:
```javascript
{
  id: string,
  name: string,
  typeface: string,
  fontFamily: string,
  description: string,
  link: string,
  category: string,
  type: 'hub' | 'specimen',
  hub: string // for specimens
}
```

## Routing Patterns

### Collections Routes
- `/collections` - Overview
- `/collections/illustrations` - Illustrations index
- `/collections/logomarks` - Logomarks index
- `/collections/motion-graphics` - Motion graphics index

### Foundry Routes
- `/foundry/specimens` - Specimen library
- `/specimen/[typeface]` - Typeface hub
- `/specimen/[typeface]/[pattern]` - Individual specimen

**Pattern**: Both use hierarchical routing with overview → category → item

## Styling Consistency

### Common Design Tokens
- `bg-surface-primary` - Page backgrounds
- `bg-container-primary` - Card containers
- `bg-surface-secondary` - Secondary surfaces
- `bg-surface-inverse` - Specimen backgrounds
- Max width: `1400px` for content containers
- Consistent padding: `px-8 py-16`

### Typography
- Headings: `kol-display-lg`, `kol-heading-xl`
- Body: `kol-mono-text`, `kol-mono-text-lg`
- Labels: `kol-label-mono-xs`, `kol-mono-sm-fine`

### Grid Layouts
- 3-column grid on desktop: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`
- Consistent gap: `gap-6`
- Responsive breakpoints

## Benefits of Shared Architecture

1. **Consistency** - Similar patterns across different content types
2. **Reusability** - Components work in multiple contexts
3. **Maintainability** - Fix once, applies everywhere
4. **Scalability** - Easy to add new collection types or foundry sections
5. **User Experience** - Familiar navigation patterns

## Future Extensions

Both architectures support future additions:

**Collections**:
- Photography collection
- Case studies
- Design explorations

**Foundry**:
- All Typefaces index
- Licensing page
- Font download system
- Variable font playground
