# 4.3.0 Collections Overview

## Purpose
Collections pages showcase creative work organized by medium: Illustrations, Logomarks, and Motion Graphics. Each collection provides a filtered, searchable gallery with consistent navigation patterns.

## Collection Types

### 4.3.1 Illustrations
Visual explorations and conceptual illustration work spanning digital, vector, and mixed media techniques.

### 4.3.2 Logomarks
Brand identity experiments and logomark designs exploring form, symbolism, and typography.

### 4.3.3 Motion Graphics
Experimental motion graphics, generative animations, and Touch Designer explorations.

### 4.3.4 Grids
Grid systems, editorial arrangements, and responsive layout explorations that document modular structures.

## Shared Components

### CollectionHero
Hero section component displaying collection title, description, and category pill.

**Props:**
- `label` - Pill label text
- `title` - Hero heading
- `description` - Description text

**Location:** `/apps/web/src/components/sections/collections/CollectionHero.jsx`

### CollectionFilters (Organism)
Filter interface with collapsible sections, category/type/year filters, and active filter management.

**Location:** `/apps/web/src/components/sections/collections/CollectionFilters.jsx`

### CollectionGrid (Organism)
Responsive grid displaying collection items using CollectionCard molecule, with built-in "load more" functionality.

**Props:**
- `illustrations` or `logomarks` - Array of items
- `initialLimit` - Items to show initially (default: 9)
- `showLoadMore` - Enable load more button (default: true)

**Location:** `@kol/ui` - `/packages/ui/src/organisms/CollectionGrid.jsx`

### CollectionCard (Molecule)
Reusable card component supporting illustrations, logomarks, and videos with hover interactions.

**Location:** `@kol/ui` - `/packages/ui/src/molecules/CollectionCard.jsx`

## Page Structure

Each collection page follows this structure:

1. **CollectionHero** - Title, description, category
2. **CollectionFilters** - Filter interface (collapsible)
3. **CollectionGrid** - Item grid with load more
4. **QuickLinksGrid** - Navigation to other collections
5. **CtaGlobal** - Call to action footer

## Navigation Pattern

All collection pages include a consistent QuickLinksGrid at the bottom linking to:
- Illustrations
- Logomarks
- Motion Graphics

This creates a circular navigation pattern allowing users to explore all collection types.

## Filtering System

### Filter Implementation
- Filters stored in Set state
- Filter format: `"filterType:filterValue"` (e.g., `"type:Digital"`, `"year:2024"`)
- Filtering uses `Array.some()` to match any active filter
- Filters reset "show all" state when changed

### Available Filter Types
- **Category** - Primary classification
- **Type** - Medium or style
- **Year** - Creation year

## Data Structure

### Common Fields
All collection items require:
- `id` - Unique identifier
- `name` or `title` - Display name
- `description` or `subtitle` - Brief description
- `category` - Primary category
- `type` - Medium/style type
- `year` - Year created

### Item-Specific Fields
- **Illustrations:** `illustrationName` (for component)
- **Logomarks:** `logoName` (for component)
- **Videos:** `videoUrl`, `thumbnailUrl`, `touchDesigner` (optional)
