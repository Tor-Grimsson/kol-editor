# Home Page Template

**Purpose:** Documentation of the homepage/marketing page template used for product/service promotion.

> The home page is the primary entry point for most users, serving as a shop window for the product or service. This template orchestrates a strategic sequence of organisms designed to convert visitors into customers, showcase value propositions, and establish credibility through social proof.

## Overview

The home page follows a **funnel-based layout** that guides users through a psychological journey:

```
Awareness → Interest → Desire → Action
```

**Page URL:** `/` (root)
**Template Location:** `apps/web/src/routes/+page.tsx`
**Content Managed:** Yes (Sanity CMS)

## Page Structure

```tsx
// Home page component structure
<BaseLayout maxWidth="full">
  <SkipLink href="#main-content" />

  {/* Above-the-fold */}
  <HomeHero />

  {/* Value demonstration */}
  <FeatureGrid />

  {/* Social proof */}
  <TestimonialCarousel />

  {/* Objection handling */}
  <FAQ />

  {/* Final conversion */}
  <CTASection />

  <SkipLink href="#main-content" />
</BaseLayout>
```

## Content Sections

### 1. Hero Section (Above-the-fold)

**Organism:** HomeHero (3.3.x)
**Priority:** Critical (must load immediately)
**Content:** Primary value proposition, primary CTA, hero visual

#### Content Structure

```tsx
interface HeroContent {
  eyebrow: string          // "Award-winning design system"
  headline: string         // "Build faster with purpose-built components"
  subheadline: string      // "Reusable, accessible, and beautifully designed"
  primaryCTA: {
    text: string          // "Get Started"
    href: string          // "/docs"
  }
  secondaryCTA: {
    text: string          // "View Examples"
    href: string          // "/work"
  }
  visual: {
    type: 'video' | 'image' | 'animation'
    src: string
    alt: string
  }
}
```

#### Hero Variants

**Variant A: Product-focused**
- Emphasis on features and benefits
- Product demo visual
- Technical audience

**Variant B: Story-focused**
- Emphasis on mission and values
- Human-centered imagery
- Broader audience

**Variant C: Results-focused**
- Emphasis on outcomes and ROI
- Data visualizations
- Business audience

### 2. Feature Grid

**Organism:** FeatureGrid (3.3.x)
**Priority:** High (load after hero)
**Content:** Key features with benefits

#### Grid Configuration

```tsx
interface Feature {
  icon: React.ComponentType
  title: string
  description: string
  details?: string
}

interface FeatureGridProps {
  title: string
  subtitle?: string
  features: Feature[]
  columns: 2 | 3 | 4  // Responsive: 2 on mobile, 3 on tablet, 4 on desktop
  layout: 'horizontal' | 'vertical'
}
```

#### Default Features

1. **Accessibility First**
   - WCAG 2.1 AA compliant
   - Screen reader tested
   - Keyboard navigation

2. **Design Tokens**
   - Consistent visual language
   - Easy theming
   - Brand flexibility

3. **Developer Experience**
   - TypeScript support
   - Well documented
   - Tree shakeable

4. **Performance**
   - Zero runtime CSS
   - Optimized bundles
   - Fast load times

### 4. FAQ Section

**Organism:** Accordion (3.3.x)
**Priority:** Medium
**Content:** Address common objections

#### FAQ Structure

```tsx
interface FAQ {
  question: string
  answer: string | {

  }

}

interface FAQSectionProps {
  title: string
  intro?: string
  faqs: FAQ[]
```

#### Common FAQ Categories

1. **Getting Started**
   - "How do I install the components?"
   - "What dependencies do I need?"
   - "Is there a free tier?"

2. **Technical**
   - "What frameworks are supported?"
   - "Can I customize the design?"
   - "Is TypeScript required?"

3. **Pricing**
   - "How much does it cost?"
   - "Are there team discounts?"
   - "What payment methods?"

4. **Support**
   - "Where can I get help?"
   - "Is there documentation?"
   - "Do you offer consulting?"

## Implementation

### Page Component

```tsx
// apps/web/src/routes/+page.tsx
import { client } from '@/lib/sanity'
import { PortableText } from '@portabletext/react'

const HOME_PAGE_QUERY = `*[_type == "homePage"][0]{...}`

export default async function HomePage() {
  const data = await client.fetch(HOME_PAGE_QUERY)

  if (!data) {
    throw new Error('Missing home page content')
  }

  return (
    <BaseLayout>
      <main id="main-content">
        {/* Hero */}
        <section className="home-hero">
          <Container maxWidth="lg">
            <div className="hero-content">
              <p className="hero-eyebrow">{data.hero.eyebrow}</p>
              <h1 className="hero-headline">{data.hero.headline}</h1>
              <p className="hero-subheadline">{data.hero.subheadline}</p>

              <div className="hero-ctas">
                <Button
                  as="a"
                  href={data.hero.primaryCTA.href}
                  size="lg"
                  variant="primary"
                >
                  {data.hero.primaryCTA.text}
                </Button>
                <Button
                  as="a"
                  href={data.hero.secondaryCTA.href}
                  size="lg"
                  variant="ghost"
                >
                  {data.hero.secondaryCTA.text}
                </Button>
              </div>
            </div>

            <div className="hero-visual">
              {data.hero.visual.type === 'video' ? (
                <video
                  src={data.hero.visual.src}
                  autoPlay
                  muted
                  loop
                  playsInline
                />
              ) : (
                <img
                  src={data.hero.visual.src}
                  alt={data.hero.visual.alt}
                  loading="eager"
                />
              )}
            </div>
          </Container>
        </section>

        {/* Features */}
        <section className="home-features">
          <Container maxWidth="lg">
            <div className="section-header">
              <h2>{data.features.title}</h2>
              {data.features.subtitle && (
                <p className="section-subtitle">{data.features.subtitle}</p>
              )}
            </div>

            <FeatureGrid features={data.features.features} />
          </Container>
        </section>

        {/* Testimonials */}
        <section className="home-testimonials">
          <Container maxWidth="lg">
            <TestimonialCarousel testimonials={data.testimonials} />
          </Container>
        </section>

        {/* FAQ */}
        <section className="home-faq">
          <Container maxWidth="md">
            <div className="section-header">
              <h2>{data.faq.title}</h2>
              {data.faq.intro && (
                <p className="section-subtitle">{data.faq.intro}</p>
              )}
            </div>

            <Accordion items={data.faq.items} />
          </Container>
        </section>

        {/* Final CTA */}
        <section className="home-cta">
          <CTASection {...data.cta} />
        </section>
      </main>
    </BaseLayout>
  )
}
```

## Analytics

### Track Events

```tsx
// Track CTA clicks
function trackCTAClick(ctaType: string, location: string) {
  gtag('event', 'cta_click', {
    event_category: 'engagement',
    event_label: `${ctaType}_${location}`,
    value: 1
  })
}

// Track scroll depth
function useScrollTracking() {
  useEffect(() => {
    const handleScroll = throttle(() => {
      const scrollPercent = (window.scrollY / document.body.scrollHeight) * 100

      if (scrollPercent >= 25 && !hasTracked25) {
        trackEvent('scroll_depth', '25%')
        setHasTracked25(true)
      }
      if (scrollPercent >= 50 && !hasTracked50) {
        trackEvent('scroll_depth', '50%')
        setHasTracked50(true)
      }
    }, 1000)

    window.addEventListener('scroll', handleScroll)
    return () => window.removeEventListener('scroll', handleScroll)
  }, [])
}
```

## Best Practices

### Do's

✅ Use clear, benefit-focused headlines
✅ Keep above-the-fold content minimal
✅ Test CTAs with real users
✅ Optimize for mobile first
✅ Use semantic HTML
✅ Implement proper heading hierarchy
✅ Track user engagement
✅ A/B test critical elements
✅ Load hero visual efficiently
✅ Provide alternative content for images

### Don'ts

❌ Use vague, generic headlines
❌ Cram too much above the fold
❌ Ignore mobile users
❌ Use stock photos with no context
❌ Skip alt text for images
❌ Mix multiple CTAs in hero
❌ Forget to test loading performance
❌ Use autoplay videos with sound
❌ Ignore accessibility guidelines
❌ Skip analytics tracking

## Common Issues

### Issue: Low conversion rate

**Solution:**
- Simplify headline to focus on single benefit
- Use more specific, action-oriented CTAs
- Add social proof near CTAs
- Reduce cognitive load

### Issue: High bounce rate

**Solution:**
- Optimize hero image load time
- Ensure mobile performance
- Test alternative hero layouts
- Add scroll indicators

### Issue: Low scroll depth

**Solution:**
- Add progress indicators
- Use shorter sections
- Include sticky navigation
- Break up long text blocks
