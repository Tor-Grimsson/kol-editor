## Overview

This document catalogs all specimen pages created for the foundry typefaces. Specimens are full-page typographic demonstrations showing typefaces in context with real content.

## Specimen Patterns

### Hub Pages
Hub pages serve as entry points for typeface-specific specimens. They typically include:
- Typeface overview
- Links to specific specimen types
- Navigation to related specimens

### Complete Specimens
"Complete" specimens are comprehensive demonstrations showing:
- Multiple weights and styles
- Character sets
- Various sizes and contexts
- Real-world usage examples

### Contextual Specimens
Contextual specimens demonstrate typefaces in specific use cases:
- Editorial (magazines, books)
- Scientific (papers, formulas)
- Legislative (legal documents)
- Data (tables, charts)
- UI (menus, forms)

### Grid Specimens
Grid specimens are enhanced versions of regular specimens with visual grid overlays:
- 12-column grid system with 24px gutters and 48px margins
- Baseline grid overlay (24px, red at 10% opacity)
- 8px subdivision grid for precise vertical rhythm (gray at 20% opacity)
- Column grid overlay (blue at 5% opacity with 20% border)
- Toggle button to show/hide grid overlays (using GridToggle component)
- Black and white color scheme for clarity
- All content aligned to grid system

**GridToggle Component** (`components/specimens/GridToggle.jsx`):
- Icon-based button positioned at bottom-right (fixed bottom-8 right-8)
- Rounded full button with 2×2 grid icon
- Uses design tokens (bg-surface-inverse, text-auto-inverse)
- Visual feedback with opacity changes
- Accessible with aria-label and title attributes

### Layout System Specimens
Layout specimens demonstrate grid systems and typographic structure:
- Grid demonstrations
- Baseline grid systems
- Multi-column layouts
- Swiss/Bauhaus design principles

### Card-Based Specimens (Refactored)
Some specimen selections use a modular card-based architecture:
- **GullhamrarSelection** - 23 card components (previously 978 lines monolithic)
- **DylgjurSelection** - 23 card components (previously 657 lines monolithic)
- **RestComplete1Selection** - 4 card components (previously 293 lines)
- **RestComplete2Selection** - 4 card components (previously 232 lines)
- **RestComplete3Selection** - 5 card components (previously 369 lines)
- **RestComplete4Selection** - 5 card components (previously 280 lines)

**Benefits of card-based architecture:**
- Each page is a separate component (20-50 lines each)
- Easy to edit, reorder, or remove individual pages
- Clear separation of concerns
- Better Git diff visibility
- Reusable components

**Card Component Pattern:**
```jsx
export default function CardName({ columns, gutter, marginX }) {
  return (
    <section className="w-full min-h-screen bg-[color]"
             style={{ paddingLeft: `${marginX}px`, paddingRight: `${marginX}px` }}>
      <div className="grid w-full"
           style={{ gridTemplateColumns: `repeat(${columns}, 1fr)`, gap: `${gutter}px` }}>
        {/* Card content */}
      </div>
    </section>
  )
}
```

**Main Selection File Pattern:**
```jsx
import { useState } from 'react'
import GridToggle from '../../../components/specimens/GridToggle'
import Card1 from './cards/Card1'
// ... all card imports

export default function Selection() {
  const [showGrid, setShowGrid] = useState(true)
  const columns = 12
  const gutter = 24
  const marginX = 48
  const baselineGrid = 24

  return (
    <div className="w-full min-h-screen relative bg-surface">
      <GridToggle showGrid={showGrid} onToggle={() => setShowGrid(!showGrid)} />

      {/* Column Grid Overlay */}
      {showGrid && (
        <div className="fixed inset-0 pointer-events-none z-40" style={{ paddingLeft: `${marginX}px`, paddingRight: `${marginX}px` }}>
          <div className="h-full grid" style={{ gridTemplateColumns: `repeat(${columns}, 1fr)`, gap: `${gutter}px` }}>
            {[...Array(columns)].map((_, i) => (
              <div key={i} className="bg-blue-500/5 border-l border-r border-blue-500/20"></div>
            ))}
          </div>
        </div>
      )}

      {/* Baseline Grid Overlay */}
      {showGrid && (
        <div className="fixed inset-0 pointer-events-none z-40" style={{
          backgroundImage: `
            repeating-linear-gradient(to bottom, rgba(239, 68, 68, 0.1) 0px, rgba(239, 68, 68, 0.1) 1px, transparent 1px, transparent ${baselineGrid}px),
            repeating-linear-gradient(to bottom, rgba(200, 200, 200, 0.2) 0px, rgba(200, 200, 200, 0.2) 1px, transparent 1px, transparent 8px)
          `
        }}></div>
      )}

      <Card1 columns={columns} gutter={gutter} marginX={marginX} />
      {/* ... all cards */}
    </div>
  )
}
```

## Typography

### Fonts Used
- **TG Rót** - Display, headings
- **TG Málrómur** - Body text, editorial
- **Torgrot** - Character demonstrations
- **Dylgjur** - Specific specimens
- **Gullhamrar** - Specific specimens
- **Silfurbarki** - Specific specimens

### Responsive Sizing
All specimens use `clamp()` for fluid typography:
```css
font-size: clamp(min, preferred, max)
/* Example: clamp(48px, 6vw, 96px) */
```

## Design Principles

1. **Swiss Typography** - Grid-based layouts, systematic spacing
2. **Icelandic Language Support** - Full character set (Á, Ð, É, Í, Ó, Ö, Þ, Æ, Ý)
3. **Baseline Grid** - Vertical rhythm alignment
4. **Responsive Design** - Fluid scaling across devices
5. **Real Content** - Actual Icelandic poetry, prose, and text

## Adding New Specimens

### Option 1: Monolithic Specimen (Simple/Small)

To add a new single-file specimen:

1. **Create component file** in appropriate directory:
   ```
   apps/web/src/routes/specimens/{typeface}/{SpecimenName}.jsx
   ```

2. **Import in App.jsx**:
   ```jsx
   import SpecimenName from './routes/specimens/{typeface}/{SpecimenName}'
   ```

3. **Add route**:
   ```jsx
   <Route path="specimen/{typeface}/{slug}" element={<SpecimenName />} />
   ```

4. **Update this documentation** with the new route

### Option 2: Card-Based Specimen (Complex/Large)

For specimens with many pages (15+ pages), use the card-based architecture:

1. **Create cards directory**:
   ```bash
   mkdir apps/web/src/routes/specimens/{typeface}/cards
   ```

2. **Create individual card components**:
   ```jsx
   // apps/web/src/routes/specimens/{typeface}/cards/PageCard.jsx
   export default function PageCard({ columns, gutter, marginX }) {
     return (
       <section className="w-full min-h-screen bg-white"
                style={{ paddingLeft: `${marginX}px`, paddingRight: `${marginX}px` }}>
         <div className="grid w-full"
              style={{ gridTemplateColumns: `repeat(${columns}, 1fr)`, gap: `${gutter}px` }}>
           {/* Page content */}
         </div>
       </section>
     )
   }
   ```

3. **Create main selection file**:
   ```jsx
   // apps/web/src/routes/specimens/{typeface}/{TypefaceSelection}.jsx
   import { useState } from 'react'
   import PageCard from './cards/PageCard'

   export default function TypefaceSelection() {
     const [showGrid, setShowGrid] = useState(true)
     const columns = 12
     const gutter = 24
     const marginX = 48
     const baselineGrid = 24

     return (
       <div className="w-full min-h-screen relative">
         {/* Grid toggle button */}
         {/* Grid overlays */}
         <PageCard columns={columns} gutter={gutter} marginX={marginX} />
       </div>
     )
   }
   ```

4. **Import and add route in App.jsx**

5. **Update this documentation**

**When to use card-based architecture:**
- 15+ pages in the specimen
- Need to frequently edit individual pages
- Want to reuse pages across different specimens
- File size exceeds 500 lines

## Notes

- All specimens should follow the design system color tokens
- Use semantic HTML for accessibility
- Consider print styles for specimen pages
- Maintain consistent navigation patterns across hub pages
