# Typography System (v3.0)

**Purpose:** Unified typography implementation with Figma design tokens and fluid responsive scaling.

> The kolkrabbi typography system is fully unified with Figma design tokens, using a 6-group structure: **Display**, **Heading**, **Text**, **Mono**, **Label**, and **Helpers**. All styles use fluid responsive sizing with `clamp()` for seamless scaling from mobile to desktop.

## Design Philosophy

- **Figma-First**: Typography styles directly map to Figma design tokens for perfect design→code translation
- **Semantic Naming**: Class names reflect purpose and hierarchy, not arbitrary sizes
- **Fluid Responsive**: All styles use `clamp()` for smooth scaling between breakpoints
- **Mix & Match**: Use any style where it works visually - hierarchy is a guide, not a rule
- **Font Variants**: Tight (extra-condensed), Narrow (condensed), and Mono properly defined with `font-stretch`
- **Harmonic Scale**: Based on research-derived 15-step scale (8, 10, 12, 14, 16, 18, 20, 24, 28, 32, 40, 48, 64, 80, 96px)

## Font Families & Variants

The system uses three core font families with explicit `font-stretch` values:

### Right Grotesk Tight
- **Font Stretch**: `extra-condensed`
- **Weight**: 500 (Medium)
- **Usage**: Display headings, uppercase statements
- **Token**: `var(--kol-font-family-rgrot-tight)`

### Right Grotesk Narrow
- **Font Stretch**: `condensed`
- **Weight**: 500 (Medium)
- **Usage**: Content headings, compact labels
- **Token**: `var(--kol-font-family-rgrot-narrow)`

### Right Grotesk Compact
- **Font Stretch**: Normal
- **Weight**: 470 (Medium)
- **Usage**: Condensed body text, subtitles, secondary content
- **Token**: `var(--kol-font-family-rgrot-compact)`

### Inter Tight
- **Font Stretch**: Normal
- **Weights**: 400 (Regular), 700 (Bold)
- **Usage**: Body copy, long-form reading
- **Token**: `var(--kol-font-family-body)`

### Right Grotesk Mono
- **Font Stretch**: Normal
- **Weights**: 500 (Medium), 300 (Fine)
- **Usage**: Code, data, technical content, labels, helpers
- **Token**: `var(--kol-font-family-mono)`

## Usage Guidelines

### Color Adaptation

Every typography class includes `color: var(--kol-surface-on-primary)` so text automatically adapts when used inside `.bg-surface-inverse` contexts. Pair with `.text-auto` utility class for explicit semantic color control.

### Responsive Strategy

All classes (except Helpers) use `clamp(min, preferred, max)` for fluid responsive scaling:
- **Mobile**: Minimum size (default viewport)
- **Desktop**: Maximum size (≥1024px viewport)
- **Fluid**: Scales smoothly between breakpoints based on viewport width

Helper classes use fixed sizes as they're primarily in buttons/UI controls where consistent sizing is preferred.

### Mix & Match Philosophy

Don't feel constrained by semantic naming - if `.kol-display-lg` works visually for body text, use it. The system provides scales and styles as building blocks; you choose what serves the design best.

### Uppercase Transformation

Classes that include `text-transform: uppercase` in their definition:
- All Display classes
- All Label classes
- All Helper classes
- `.kol-heading-sm`
- All Heading Tight classes (`.kol-heading-tight-*`)

Other classes can be combined with Tailwind's `.uppercase` utility when needed.

## Implementation Tips

1. **Use atoms first**: Components like `<SectionHeader>` and `<Tag>` already map the correct classes. Reach for raw classes only when building new primitives.

2. **Pair with `.text-auto`**: Ensures automatic contrast adjustments across light/dark and inverse surfaces.

3. **Respect fluid scaling**: Avoid hardcoding font sizes - let the `clamp()` baked into classes handle device differences.

4. **Font-stretch is crucial**: When customizing, always include `font-stretch: extra-condensed` or `font-stretch: condensed` for Right Grotesk variants.

## Prose Typography System

For rich-text and long-form content, use one of three prose wrapper classes:

### Prose Variants

| Class | Max Width | Usage |
|-------|-----------|-------|
| `.kol-prose` | 65ch (~1040px) | Standard article width, balanced readability |
| `.kol-prose-wide` | 90ch (~1440px) | Large-scale typography for foundry pages, hero content |
| `.kol-prose-compact` | 45ch (~720px) | Narrow reading column for sidebars, cards |

**Features:**
- Full semantic HTML support: `h1-h4`, `p`, `a`, `blockquote`, `ul`, `ol`, `code`, `pre`, `hr`
- Fluid typography using `clamp()` for responsive scaling
- Logical properties (`block`/`inline`) for i18n support
- Integrated with design system color and font-family tokens
- Theme-aware with automatic light/dark mode adaptation

**Example:**
```html
<article class="kol-prose">
  <h1>Article Title</h1>
  <p>Body content with automatic styling...</p>
  <blockquote>Pull quotes with accent border</blockquote>
</article>
```

## Migration from v2.x

If you're upgrading from v2.x:

1. **Display group is new**: Map old `kol-heading-display` → `kol-display-lg`
2. **Helpers group is new**: Use for button text, CTA copy, utility labels
3. **Font-stretch added**: All Right Grotesk classes now have explicit `font-stretch`
4. **Tablet breakpoint removed**: System now uses mobile→desktop only (simplified from 3→2 breakpoints)
5. **New classes**: `kol-text-lg`, `kol-mono-sm`, label variants, all helper classes

## Technical Implementation

Typography styles are defined in:
- **Classes**: `packages/ui/css/components.css`
- **Font Tokens**: `packages/ui/theme.css`
- **Documentation**: `docs/documentation/2.2.0-design-system-typography.md` (this file)
- **Figma Source**: `_import/tokens-type.json`

Font files are loaded via `@font-face` definitions in `theme.css`. Apps consuming `@kol/ui` automatically get all typography styles.
