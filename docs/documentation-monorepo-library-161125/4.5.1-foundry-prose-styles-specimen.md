## Design Philosophy

### The Problem

Traditional type specimens often focus on:
- Abstract letterform displays
- Character sets and glyphs
- Weight and style variations in isolation
- Lorem ipsum placeholder text

These approaches, while beautiful, don't demonstrate **how the typeface performs in real-world contexts**.

### The Solution

Instead of showing "what the typeface looks like," we demonstrate **"what the typeface can do"** by implementing actual use cases:

- Editorial publishing
- Scientific data presentation
- Restaurant hospitality design
- Academic newsletters
- Reference materials
- Book design

This approach transforms the specimen from a showcase into a **functional design system** that designers can reference when implementing similar layouts.

## Technical Architecture

### Layout Structure

```jsx

  {/* 8 distinct sections, each full-screen or larger */}
  <section>...</section>
  <section>...</section>
  // ...
</div>
```

### Key Design Decisions

1. **Cream background** (`#F5F1E8`) - Warm, paper-like feel that reduces eye strain
2. **Full-width responsive** - Not constrained to fixed width like print layouts
3. **Section-based scrolling** - Each prose style is its own contained environment
4. **Single typeface throughout** - TG_Malromur in various weights/sizes
5. **Real content** - No lorem ipsum; authentic text for each context

## The Eight Sections

### 1. Title Page

**Purpose**: Elegant entry point establishing the specimen's purpose

**Typography**:
- 64px main title
- Italic subtitle
- Horizontal rules for visual separation
- Small foundry attribution

**Design Pattern**:
```jsx
<section className="w-full min-h-screen flex items-center justify-center">
  <div className="max-w-[640px] mx-auto text-center">
    {/* Centered vertical alignment with generous spacing */}
  </div>
</section>
```

**Key Insight**: The title page uses the typeface's medium weight with generous tracking to create elegance without fragility.

---

### 2. Table of Contents

**Purpose**: Demonstrate typographic navigation with leader dots

**Typography**:
- 18px entry labels
- Dotted border leaders
- Right-aligned page numbers
- Nested hierarchy with indentation

**Design Pattern**:
```jsx
<div className="flex justify-between items-baseline">
  <span>Editorial</span>
  <span className="flex-1 border-b border-dotted border-black mx-4 mb-1"></span>
  <span>2</span>
</div>
```

**Technical Solution**: Using flexbox with `flex-1` on the leader creates automatic spacing that adapts to content width.

**Key Insight**: Leader dots are achieved via CSS border, not actual dots, making them responsive and preventing orphaned dots.

---

### 3. Editorial Article

**Purpose**: Multi-column magazine/journal layout with sidebar

**Typography**:
- 64px display headline (stacked)
- 16px body text with proper leading (line-height: 1.5)
- 14px pull quotes
- 12px sidebar captions
- Mixed weights: medium for headlines, normal for body

**Layout Structure**:
```jsx
<div className="grid grid-cols-12 gap-8">
  <div className="col-span-7"> {/* Main article */} </div>
  <div className="col-span-5 bg-[#E8DCC0]"> {/* Sidebar */} </div>
</div>
```

**Color Palette**:
- Cream background: `#F5F1E8`
- Sidebar accent: `#E8DCC0` (darker cream)
- Text: Black

**Key Insight**: The 7-5 column split creates visual interest while maintaining optimal reading line length (45-75 characters).

---

### 4. Data Table

**Purpose**: Scientific/technical data presentation

**Typography**:
- 12px uppercase headers (semibold, tracking increased)
- 14px table data
- 10px footnote/attribution
- Italic for location names in alternating rows

**Table Structure**:
```jsx
<table className="w-full border-collapse">
  <thead>
    <tr className="border-t-2 border-b border-black">
      {/* Headers with text alignment classes */}
    </tr>
  </thead>
  <tbody>
    <tr className="border-b border-gray-300">
      {/* Data cells with proper alignment */}
    </tr>
  </tbody>
</table>
```

**Data Alignment**:
- Left: Location names
- Center: Individual measurements
- Right: Calculated averages (semibold)

**Key Insight**: Proper alignment in tables is crucial - text left, numbers right/center, with bold for totals/averages.

---

### 5. Menu / Bill of Fare

**Purpose**: Restaurant/hospitality design

**Typography**:
- 12px uppercase section headers (semibold, letter-spacing)
- 14px menu items
- 10px wine list (two-column grid)
- Mix of roman and italic for variety

**Layout Pattern**:
```jsx
<div className="grid grid-cols-2 gap-x-16 gap-y-12">
  <div className="space-y-8"> {/* Left column */}
    <div> {/* Category */}
      <h3>Soups.</h3>
      <div className="space-y-2 text-center">
        {/* Items */}
      </div>
    </div>
  </div>
</div>
```

**Hierarchy**:
1. Section headers (Soups, Fish, Game, etc.)
2. Category items
3. Subcategories (indented or italic)
4. Special sections (Dessert, Wines) with decorative rules

**Key Insight**: Restaurant menus benefit from tight leading and generous category spacing. Centered alignment creates formality.

---

### 6. Newsletter / Bulletin

**Purpose**: Academic or scientific publication

**Typography**:
- 48px main title
- 24px italic subtitle
- 20px article headline
- 16px italic pullout quote
- 14px three-column body text

**Layout Structure**:
```jsx
<div className="columns-3 gap-8">
  <p className="mb-4">
    {/* Auto-flows into columns */}
  </p>
</div>
```

**CSS Columns**: Using native CSS multi-column layout for automatic text flow across columns.

**Key Insight**: CSS columns are better than grid for continuous text - they handle text flow naturally without manual breaking.

### 7. Index / Directory

**Purpose**: Reference material with dense information

**Typography**:
- 14px primary terms (semibold)
- 12px definitions (italic, indented)
- Three-column layout
- Vertical rhythm via `space-y-3`

**Pattern**:
```jsx
<div className="columns-3 gap-8">
  <div className="space-y-3">
    <div>
      <p className="font-semibold">√Åst</p>
      <p className="pl-4 text-xs italic">love, affection</p>
    </div>
  </div>
</div>
```

**Key Insight**: Dictionaries and indexes need clear visual hierarchy through bold/regular pairing and consistent indentation.

---

### 8. Two-Page Spread / Chapter Opening

**Purpose**: Book interior page design

**Typography**:
- 12px uppercase chapter label
- 36px chapter title (semibold)
- 16px italic opening quote
- 16px body text with generous leading

**Layout Structure**:
```jsx
<div className="grid grid-cols-2 gap-16">
  <div className="flex items-center justify-end">
    {/* Left page - chapter title, right-aligned */}
  </div>
  <div className="flex items-center">
    {/* Right page - body text, left-aligned */}
  </div>
</div>
```

**Key Insight**: The two-page spread uses vertical centering and asymmetric text placement to simulate book gutter and margins.

## Typography System

### Font Syntax

All typography uses the custom font syntax:
```jsx
className="font-['TG_Malromur']"
```

### Weight Scale Used

- `font-normal` (400) - Body text, captions
- `font-medium` (500) - Subheadings
- `font-semibold` (600) - Section headers, chapter titles
- `font-bold` (700) - Primary headlines, table headers

### Size Scale

| Context | Size | Usage |
|---------|------|-------|
| Display | 48-64px | Main headlines |
| Headline | 24-36px | Article titles, chapter names |
| Subhead | 18-20px | Section headers |
| Body | 14-16px | Primary reading text |
| Caption | 12-14px | Table headers, footnotes |
| Fine Print | 10-12px | Attributions, legal text |

### Leading (Line Height)

- Display text: 1.1-1.2
- Headlines: 1.2-1.3
- Body text: 1.4-1.5
- Dense listings: 1.3-1.4

## Lessons Learned

### What Worked Well

1. **Real content approach** - Made the specimen immediately useful

1. **Leader dot implementation** - CSS borders vs. actual dots
